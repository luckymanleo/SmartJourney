# 行程地图交互升级方案

> 日期: 2026-06-08 | 状态: 方案，未实施

---

## 当前问题

现在的 TripMap 是「行程总览图」：把所有天的 POI 丢到地图上，按天着色。

用户实际需求是**出行中的导航工具**：
- "今天要去哪几个地方？按什么顺序走？"
- "这个景点长什么样？值不值得绕路去？"
- "从酒店到景点怎么走？走过去要多久？"

---

## 方案设计

### 交互模型

```
┌──────────────────────────────┐
│ 🗺️ 第1天  ▼  切换天            │  ← 天选择器
├──────────────────────────────┤
│                              │
│   🏨 酒店 ──→ 🎫 景点A       │  ← 路线连线
│                │             │
│                ↓             │
│             🍽️ 餐厅          │  ← 当前选中 POI 高亮
│                              │
├──────────────────────────────┤
│ 点击 POI → 底部详情卡片       │  ← 双面板
└──────────────────────────────┘

┌──────────────────────────────┐
│ 📷 故宫                         │  ← 详情面板
│ ⭐ 4.8 · 世界文化遗产           │
│ [图1] [图2] [图3] ...          │  ← 景点图片
│ 🚶 步行 15分钟 → 下一站         │  ← 导航到下一站
│ [开始导航]                      │  ← 唤起高德导航
└──────────────────────────────┘
```

### 核心功能

#### 1. 天选择器

- 顶部下拉或 Tab：`第1天 ▼` → 选择某一天
- 地图只显示当天的 POI + 路线
- 「全部」选项保留总览

#### 2. 路线连线

- 当天 POI 按 `sort_order` 画折线（步行路线实线 / 驾车虚线）
- 线段颜色随进度渐变（已走=灰，当前=蓝，未走=浅蓝）
- 点击线段 → 显示距离和时间

#### 3. POI 详情卡片 + 图片

- 点击地图标记 → 底部滑出详情卡片
- 图片来源：
  - **高德 POI 详情 API** → 返回 `photos` 字段（实景图）
  - **备选**：百度图片搜索、Unsplash
- 卡片内容：名称 / 评分 / 类型 / 图片轮播 / 「去这里」按钮

#### 4. 导航唤起

- 每个 POI 卡片底部「开始导航」按钮
- 点击 → 唤起高德地图 App（DeepLink）或打开 Web 导航
- URL scheme: `https://uri.amap.com/navigation?to={lng},{lat},{name}&mode=walking&callnative=1`

#### 5. 手机版 vs PC 版差异

| 功能 | 手机版 | PC 版 |
|------|--------|-------|
| 地图占比 | 全屏 | 主区域 70% + 侧栏 30% |
| 详情卡片 | 底部 Sheet | 右侧固定面板 |
| 图片轮播 | 横滑 | 网格 |
| 导航按钮 | 「开始导航」FAB | 「去这里」按钮 |

## 数据依赖

### 需要新增的后端能力

| API | 用途 | 优先级 |
|-----|------|--------|
| `GET /api/v1/map/trip/{id}/day/{n}` | 单天地图数据（POI + 坐标） | P0 |
| `GET /api/v1/map/poi/{id}` | POI 详情（已有，含 `photos` 字段） | P0 |
| `GET /api/v1/map/direction?from=...&to=...&mode=walking` | 两点路线（已有） | P0 |
| POI 图片接口 | 景点实景图（高德 `photos` 字段已有） | P0 |

### 已有能力（直接复用）

- `GET /api/v1/map/trip/{id}` — 行程聚合（含所有天 POI）
- `GET /api/v1/map/poi/{id}` — POI 详情（含 `photos`）
- `GET /api/v1/map/direction` — 路径规划
- `GAODE_API_KEY` — JS SDK 和 REST API

## 分阶段实施

### P0 — 单天地图 + POI 详情

| 项目 | 说明 | 改动 |
|------|------|------|
| `day_selector` | 天选择器下拉 | TripMap 新增 prop/state |
| `poi_detail_panel` | 底部/右侧详情卡片 | 新组件 `PoiDetailCard` |
| `route_lines` | 当天 POI 连线 | 调用 `direction` API 或简单直线 |
| `photos` | POI 图片展示 | 复用 `poi_detail` API 的 `photos` 字段 |

### P1 — 导航唤起 + 路线优化

| 项目 | 说明 | 改动 |
|------|------|------|
| `navigate_button` | 唤起高德导航 | DeepLink URL |
| `route_style` | 路线类型区分（步行/驾车/公交） | 不同线型/颜色 |

### P2 — 增强图片

| 项目 | 说明 |
|------|------|
| `image_fallback` | 高德无图时用搜索引擎补充 |
| `image_gallery` | 图片横滑/灯箱查看 |

## 关键交互细节

1. **默认选中**：进入地图 Tab 时默认选中"第1天"
2. **POI 点击**：地图标记点击 → 详情卡片滑出 + 地图中心移动到该 POI
3. **切换天**：选择其他天 → 清空详情卡片 + 重新渲染该天 POI + 适配视野
4. **导航按钮**：手机版唤起高德 App，PC 版打开高德 Web 导航页
5. **图片加载**：优先高德 `photos`，无图时显示默认占位图
