# SmartJourney PC Web 版设计方案

> 2026-06-03 | 在现有移动端前端基础上，新增 PC Web 版本，现有代码零改动。

## 核心策略

**Multi-page Vite 构建** — 同一 `npm run build` 产出两个版本：
- 移动端：`index.html` → `src/main.tsx` → `App.tsx`（现有，不改）
- PC 端：`pc.html` → `src/main-pc.tsx` → `AppPC.tsx`（新建）

所有 Store、API、子组件（SearchCards、TripTimeline、TripCard、BudgetPanel、CityCascader）均为共享，不复制。

## 实现阶段

### Phase 1: 基础设施（LayoutPC + AppPC + vite 配置）

- pc.html, main-pc.tsx, AppPC.tsx, LayoutPC.tsx, index-pc.css
- Sidebar 导航（复用现有路由路径）
- Header 栏（用户信息、退出登录）
- vite.config.ts 多入口配置

### Phase 2: 页面迁移（按复杂度排序）

1. **SettingsPagePC** — 最简单，扩宽表单即可
2. **MyTripsPagePC** — TripCard 网格化
3. **HomePagePC** — 双栏布局
4. **SearchPagePC** — 水平搜索栏 + 网格结果
5. **TripDetailPagePC** — 双栏时间线
6. **PlanPagePC** — 左右分栏（最复杂）

每个页面：
- 共享 Store（authStore, searchStore, planStore, tripStore）
- 共享 API（api/index.ts）
- 共享子组件（SearchCards, TripTimeline, TripCard, BudgetPanel, CityCascader）
- 仅重写页面布局/容器

---

## 技术要点

1. **响应式边界**：PC 布局使用 `lg:` 断点（1024px），小屏回退到移动端样式
2. **IndexedDB 无影响**：搜索历史和 Zustand 状态在两个版本间天然隔离（不同浏览器标签页）
3. **CityCascader**：PC 版可改为 Popover 而不是 bottom sheet（`absolute` 定位，跟随输入框）
4. **TripTimeline**：PC 版宽度更宽，时间线可展开更多 detail
5. **不复制业务逻辑**：所有 API 调用、状态管理、数据解析零重复

## 不做什么

- ❌ 不改动任何现有文件（pages/、components/、stores/、api/、App.tsx、Layout.tsx、main.tsx、index.html、index.css）
- ❌ 不创建独立 npm 包或 monorepo
- ❌ 不引入新依赖（已有的 react, zustand, tailwind, lucide-react 足够）
- ❌ 不添加后端路由或 API 变更
