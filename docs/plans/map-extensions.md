# 地图功能扩展应用分析

> 日期: 2026-06-09 | 状态: 分析，未实施

---

## 当前应用

地图仅用于**行程详情页**——展示已生成行程的 POI 位置和天路线。

---

## 可扩展场景

### 1. 搜索结果地图（★★★★★ 最高收益）

**场景**：搜索酒店/景点时，结果卡片只显示文字，用户无法判断位置优劣。

```
当前搜索页                    加入地图后
┌──────────────────┐    ┌──────────┬───────────────┐
│ 🏨 酒店A ¥288     │    │ 结果列表   │               │
│ 🏨 酒店B ¥350     │    │ 🏨 A ¥288  │   🗺️ 地图     │
│ 🏨 酒店C ¥420     │    │ 🏨 B ¥350  │  A ●  B ●    │
│ ...              │    │ 🏨 C ¥420  │       C ●     │
└──────────────────┘    │ ...       │               │
                        └──────────┴───────────────┘
                        移动：上下分屏  PC：左右分栏
```

| 搜索类型 | 地图展示 | 数据来源 |
|---------|---------|---------|
| 机票 | 无意义（城市间飞行） | — |
| 火车 | 无意义（城市间铁路） | — |
| 酒店 | ✅ 标记位置 + 价格气泡 | MCP 搜索结果含坐标时可用 |
| 景点 | ✅ 标记位置 + 评分/门票 | geocode 景点名 |
| 美食 | ✅ 标记位置 + 人均 | geocode 餐厅名 |
| 交通 | ✅ 画出路线 | direction API |

**改动**：`SearchCards.tsx` 新增 `SearchMap` 子组件，`SearchPage.tsx` / `SearchPagePC.tsx` 加右侧地图栏。

### 2. AI 规划实时地图（★★★★★ 高收益）

**场景**：AI 规划进行中（SSE 流式返回），用户只能看文字进度条，无法感知地理位置。

```
规划中：实时地图逐一亮起                   完成后：完整行程预览
┌──────────────────────┐          ┌──────────────────────┐
│ 🗺️ 正在规划...        │          │ 📍 行程预览            │
│                      │          │                      │
│  ● 北京 (出发)        │   →      │ [保存行程] [重新规划]   │
│  ○ 天津 (目的地)       │          │                      │
│  ○ 酒店 (搜索中...)    │          │ ● 北京 → ● 天津       │
│                      │          │  ○ 酒店  ○ 景点A      │
└──────────────────────┘          └──────────────────────┘
```

**数据流**：
```
SSE: tool_call(search_hotel, "天津酒店")
  → 搜索结果返回 → geocode 酒店地址 → 地图上亮起对应标记
SSE: tool_call(search_poi, "天津景点")
  → 搜索结果返回 → geocode 景点 → 地图标记
...
SSE: trip_data {days: [{items: [...]}]}
  → 完整行程坐标 → 地图显示完整路线
```

**改动**：`PlanPage.tsx` 中 SSE 事件处理增加地图更新逻辑，`PlanPagePC.tsx` 加右侧实时地图。

### 3. 行程列表全局地图（★★★★ 中收益）

**场景**：用户有多个行程（去过三亚/去过成都/计划去天津），需要一个总览视角。

```
我的行程 — 地图总览
┌──────────────────────────────┐
│ [列表] [🗺️ 地图总览]          │
│                              │
│       🗺️ 中国地图             │
│                              │
│   🔵 三亚 (已完成)             │
│   🟢 成都 (已完成)             │
│   🟠 天津 (规划中)             │
│                              │
│  点击标记 → 跳转该行程详情      │
└──────────────────────────────┘
```

**数据**：每个行程有 `origin` 和 `destination` 字段，geocode 后即可标记。

**改动**：`MyTripsPage.tsx` / `MyTripsPagePC.tsx` 新增 "地图总览" Tab。

### 4. 规划页目的地预览（★★★ 中收益）

**场景**：用户输入目的地后，在开始规划前可以先在地图上看看目的地在哪。

```
规划页
┌──────────────────────────────┐
│ 📍 目的地：天津                │
│ 🗺️ [城市预览]                  │
│                              │
│  天津站 ●  天津之眼 ●          │
│  五大道 ●  古文化街 ●          │
│                              │
│ [输入详情后开始规划]            │
└──────────────────────────────┘
```

**改动**：`PlanPage.tsx` 在填写目的地后展示小型预览地图。

### 5. 热门目的地地图（★★★ 中收益）

**场景**：首页热门目的地是卡片列表，加入地图展示更直观。

```
首页
┌──────────────────────────────┐
│ 🗺️ 热门目的地                  │
│                              │
│  🏖️三亚  🐼成都  🏯西安        │
│  🌊杭州  🏔️大理  🌺厦门         │
│                              │
│ [地图视图]                     │
└──────────────────────────────┘
```

**改动**：`HomePage.tsx` / `HomePagePC.tsx` 热点门目的地卡片加地图预览。

## 推荐实施顺序

```
Phase 1: 搜索结果地图 (酒店+景点+美食)
  → 搜索体验质的提升
  → 复用 SearchCards 的搜索结果数据
  → 约 1d

Phase 2: AI 规划实时地图
  → 规划过程可视化
  → 需要在 SSE 流中增加坐标数据
  → 约 1.5d

Phase 3: 行程列表地图总览
  → 全局视角
  → 低实现成本
  → 约 0.5d
```
