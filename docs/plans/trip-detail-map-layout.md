# 行程详情页地图布局优化方案

> 日期: 2026-06-08 | 状态: ✅ 已实施

---

## 当前问题

### 手机版

```
┌─────────────────────┐
│ ← 返回  标题   🗑️    │  Header
│ 日期 · 人数 · 预算     │  Info
├─────────────────────┤
│                     │
│     🗺️ 地图 (224px)   │  ← ⚠️ 占据首屏黄金位置
│                     │     推挤核心内容 (timeline)
├─────────────────────┤
│ 🌤️ 天气              │
│                     │
│ 第1天  ✈️ 航班...    │  ← 用户想立刻看到的内容
│ 第2天  🎫 景点...    │     却被挤到屏幕下方
│ ...                  │
└─────────────────────┘
```

**问题**：用户打开行程详情页是为了看"今天去哪、几点出发"，而不是看地图。地图放在中段推挤了核心 timeline。

### PC 版

```
┌──────────────────────┬──────────────────────┐
│                      │ 🌤️ 天气              │
│   🗺️ 地图 (h-80)     │ 行程信息              │
│                      │ 💰 预算              │
├──────────────────────┤                      │
│                      │                      │
│  第1天  ✈️ 航班...    │                      │
│  第2天  🎫 景点...    │                      │
│  ...                 │                      │
└──────────────────────┴──────────────────────┘
```

**问题**：地图在左列顶部占位过大 (h-80 = 320px)，timeline 被迫下移。右列信息面板短，造成左右视觉不均衡。地图和 timeline 是竞争关系而非互补关系。

---

## 方案对比

### 方案 A：Tab 切换（✅ 推荐）

```
手机版                           PC 版
┌─────────────────────┐    ┌──────────┬──────────┐
│ ← 返回  标题   🗑️    │    │ ← 返回列表           │
│ [ 行程 ] [ 🗺️ 地图 ] │    │ [ 行程 ] [ 🗺️ 地图 ] │
├─────────────────────┤    ├──────────┴──────────┤
│                     │    │                      │
│  第1天  ✈️ 航班...    │    │   (全宽)              │
│  第2天  🎫 景点...    │    │   对应 tab 内容        │
│  ...                 │    │                      │
└─────────────────────┘    └──────────────────────┘

点击"地图" → 全屏地图视图（带图例、可交互）
```

| 维度 | 评价 |
|------|------|
| 信息密度 | ⭐⭐⭐ 每屏专注一件事 |
| 实现成本 | ⭐⭐⭐ 一个 tab state 切换 |
| 地图体验 | ⭐⭐⭐ 全宽展示，视野好 |
| 用户心智 | ⭐⭐⭐ "行程"和"地图"是两个独立视角 |

## 推荐：方案 A（Tab 切换）

### 理由

1. **心智模型清晰**："行程"看时间线，"地图"看空间分布，两个维度互补不冲突
2. **信息不推挤**：各自独占全宽，不需要在有限空间内竞争
3. **实现简单**：一个 `activeTab` state + 条件渲染，PC/手机同一套逻辑
4. **地图体验好**：全宽渲染，视野开阔，图例可完整展示
5. **扩展性好**：未来可加第 3 个 Tab（如"预算分析"）

### 改动点

| 页面 | 改动 |
|------|------|
| `TripDetailPage.tsx` | 在 Header 下方增加 Tab bar；移除当前位置的地图；Tab="地图"时渲染 `<TripMap>` |
| `TripDetailPagePC.tsx` | 同手机版逻辑；Tab 放在 Header 行右侧或下方 |
| `TripMap.tsx` | 不需要改动（compact prop 仍可根据 Tab 模式自适应） |

### 预期效果

- 手机默认打开"行程"Tab，用户直接看到 timeline
- PC 同样默认"行程"，地图可按需切换到全宽视图
- 图例在 Tab 模式下放在地图下方而非和 Timeline 竞争空间
